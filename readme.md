# Tab Groups

在 VS Code 里把文件按「分组」整理起来——不管文件当前有没有打开，都能一键找到、批量打开或关闭。

---

## 它能做什么

- **手动分组**：自己挑选文件，放进不同分组（例如「前端页面」「接口文档」）
- **正则自动分组**：用规则自动扫描项目里的文件，适合按目录或后缀批量归类
- **一键打开 / 关闭**：右键分组，一次打开或关闭该组里的所有文件
- **标签页快捷操作**：在编辑器标签上右键，快速「加入分组」或「取消分组」
- **键盘快捷键**：四条可自定义命令（加入/取消分组、新建/删除分组），默认按键见下文
- **配置保存在项目里**：分组数据在 `.vscode/tab-groups.json`，快捷键偏好在 `.vscode/settings.json`，均可随项目提交

---

## 快速开始

1. 安装扩展 **Tab Groups**（发布者：Rita）
2. 用 VS Code **打开一个文件夹**作为工作区（需为单根工作区，见下方说明）
3. 点击左侧活动栏的 **Tab Groups** 图标，打开「标签分组」侧边栏
4. 点击侧边栏顶部的 **＋** 或右键空白处，**新建分组**
5. 在编辑器中打开某个文件，在**标签页上右键** → **加入分组**，选择目标分组

分组和文件会显示在侧边栏树视图中，点击文件名即可打开。

## 界面说明

侧边栏大致如下：

```
📁 我的手动分组（手动）
   📄 src/index.ts
   📄 src/utils.ts
📁 后端逻辑（引用：backend-regex）
   📄 server.js
📁 前端组件（正则）
   📄 components/Button.tsx
```

- 分组名称后的括号表示分组类型：**手动**、**正则**，或引用的全局配置名
- 文件若已不存在（被移动或删除），会以灰色显示并标注「（不存在）」，可右键移除

## 常用操作

### 侧边栏 · 分组

| 操作 | 说明 |
|------|------|
| 新建分组 | 侧边栏标题栏 **＋**，或右键空白处；也可用快捷键（见下文） |
| 重命名 / 删除 | 右键分组名称；删除分组也可用快捷键（见下文） |
| 自定义快捷键 | 侧边栏标题栏 **键盘图标**，打开设置页录入并保存 |
| 打开组内所有文件 | 右键分组 → **打开组内所有文件**（在编辑器中批量打开） |
| 关闭组内所有文件 | 右键分组 → **关闭组内所有文件**（关闭该组已打开的标签页） |
| 设置为手动 | 改为手动维护文件列表 |
| 设置正则（内嵌） | 为当前分组单独设置匹配规则 |
| 引用全局配置 | 复用已在配置文件中定义好的规则 |
| 扫描文件 | 仅正则分组可用，按规则重新扫描并更新文件列表 |

> **说明**：「打开 / 关闭组内所有文件」针对的是**编辑器标签页**，不是侧边栏树节点的展开/折叠。

### 侧边栏 · 文件

| 操作 | 说明 |
|------|------|
| 打开文件 | 单击文件名，或右键 → **打开文件** |
| 从分组中移除 | 右键文件 → **从分组中移除** |
| 复制路径 | 右键文件 → **复制路径** |

### 编辑器标签页

在文件标签上右键：

- **加入分组** — 将当前文件加入所选分组（同一文件可属于多个分组）
- **取消分组** — 从所选分组中移除当前文件；列表首项 **全部分组** 可一次性从所有分组中移除

关闭编辑器标签**不会**把文件从分组里删掉；需要显式「取消分组」或从侧边栏移除。

## 键盘快捷键

侧边栏标题栏点击 **键盘图标** 可打开「自定义快捷键」页面，点击快捷键框后按下组合键录入，保存后写入 `.vscode/settings.json` 并同步到本机 keybindings，使快捷键立即生效。

**默认快捷键**（可在设置页修改）：

| 功能 | 默认按键 | 说明 |
|------|----------|------|
| 加入分组 | `Ctrl+Shift+I` | 需打开工作区内的文件 |
| 取消分组 | `Ctrl+Shift+O` | 需打开工作区内的文件 |
| 新建分组 | `Ctrl+Shift+U` | 需单根工作区 |
| 删除分组 | `Ctrl+Shift+P` | 需单根工作区；未选中侧边栏分组时会弹出列表让你选择 |

> **提示**：`Ctrl+Shift+P` 与 VS Code 命令面板默认快捷键相同，若冲突请在自定义页面改绑删除分组快捷键。

保存自定义快捷键需要已打开**单根工作区**；无工作区时可打开设置页预览，但无法保存。

## 三种分组方式

### 1. 手动分组（默认）

新建分组后，通过「加入分组」或侧边栏操作逐个添加文件，适合固定、少量文件的集合。

### 2. 正则分组（内嵌）

右键分组 → **设置正则（内嵌）**，输入正则表达式，再 **扫描文件**，扩展会搜索整个工作区并更新该分组的文件列表。

示例：匹配所有 `components` 目录下的 `.tsx` 文件：

```
.*/components/.*\.tsx$
```

### 3. 全局配置（多分组复用）

若多个分组要用同一套规则，可在 `.vscode/tab-groups.json` 的 `configs` 里定义全局配置，再在分组上 **引用全局配置**。

右键分组 → **管理全局配置** 可打开配置文件进行编辑。全局配置需手动在 JSON 中新增；分组侧通过「引用全局配置」选择已有项。

配置文件示例：

```json
{
  "groups": [
    {
      "id": "group-1",
      "name": "我的手动分组",
      "files": ["src/index.ts", "src/utils.ts"]
    },
    {
      "id": "group-2",
      "name": "后端逻辑",
      "files": [],
      "configId": "backend-regex"
    }
  ],
  "configs": [
    {
      "id": "backend-regex",
      "type": "regex",
      "regex": ".*/server/.*\\.js$",
      "description": "后端 JS 文件"
    }
  ]
}
```

修改并保存该文件后，扩展会自动重新加载，无需重启。

## 使用限制与提示

| 情况 | 说明 |
|------|------|
| 单根工作区 | 目前仅支持打开**一个文件夹**的工作区；多根工作区或未打开文件夹时，功能不可用 |
| 文件被移动/重命名 | 分组里存的是相对路径，文件移动后会显示为不存在，需手动从分组移除或重新加入 |
| 正则扫描 | 会**覆盖**该分组当前的文件列表；扫描大量文件时会有进度提示，可取消 |
| 删除分组 | 若该分组引用的全局配置已无其他分组使用，会询问是否一并删除该配置 |
| 快捷键配置 | 保存在 `.vscode/settings.json`；同步 keybindings 时可能移除该文件中的 JSON 注释 |
| 默认删除快捷键 | `Ctrl+Shift+P` 可能与命令面板冲突，建议自行改绑 |

## 常见问题

**Q：分组数据存在哪里？**  
A：当前工作区下的 `.vscode/tab-groups.json`。快捷键偏好存在 `.vscode/settings.json` 的 `tabGroups.shortcuts` 中。

**Q：换电脑或同事能共用分组吗？**  
A：可以。将 `tab-groups.json` 和（如需统一快捷键）`settings.json` 中的 `tabGroups.shortcuts` 一并纳入版本控制即可。每位开发者保存快捷键时仍会写入各自本机的 keybindings 文件。

**Q：正则写错了怎么办？**  
A：保存前扩展会校验；无效正则会提示错误，请检查转义（如 `.` 写成 `\.`）。

**Q：和 VS Code 自带的 Tab Groups 冲突吗？**  
A：不冲突。本扩展管理的是「文件逻辑分组」，与编辑器窗口布局里的 Tab Groups 是不同概念。

**Q：取消分组里的「全部分组」是什么？**  
A：当文件属于多个分组时，选「全部分组」会一次性从所有相关分组中移除该文件，无需逐个选择。
